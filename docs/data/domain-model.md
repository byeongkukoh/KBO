# Domain Model

## Core Entities

- Season: KBO 시즌 단위. 연도, 시즌 상태(진행 중/종료), 시작일, 종료일 등의 정보를 가진다.
- Team: KBO 구단 단위. 팀명, 약칭, 시즌별 소속 기록과 연결된다.
- Player: 선수 단위. 이름, 현재 소속 팀, 포지션, 생년월일, 나이 계산 기준 정보, 등번호 등 기본 프로필 정보를 가진다.
- PlayerTeamHistory: 선수가 어떤 시즌 또는 기간에 어떤 팀에 소속되었는지 나타내는 이력 엔티티다.
- Game: 개별 경기 단위. 경기일, 홈/원정 팀, 경기 결과, 시즌 정보를 가진다.
- InningScore: 경기의 회차별 득점 및 공격 흐름을 나타내는 엔티티다.
- PlayEvent: 경기 중 발생한 타석 결과, 득점 장면, 투수 교체, 주자 진루 등 raw 이벤트를 기록하는 엔티티다.
- PlayerSeasonStat: 특정 시즌에서 한 선수의 누적 기록을 나타낸다. 타자/투수 공통 필드와 역할별 세부 지표를 포함할 수 있다.
- TeamSeasonStat: 특정 시즌에서 한 팀의 누적 기록을 나타낸다.
- PlayerGameStat: 특정 경기에서 한 선수의 기록을 나타낸다.
- TeamGameStat: 특정 경기에서 한 팀의 기록을 나타낸다.
- DerivedPlayerMetric: OPS, wRC+, ERA+, FIP 같은 계산형 또는 고급 지표를 저장하거나 캐시하는 엔티티 또는 뷰 계층이다.
- DataSyncLog: 스크래핑 및 데이터 반영 시점과 성공 여부를 기록한다.

## Entity Relationships

- 하나의 Season은 여러 TeamSeasonStat, PlayerSeasonStat, Game과 연결된다.
- 하나의 Team은 여러 시즌에 걸쳐 TeamSeasonStat을 가진다.
- 하나의 Player는 여러 PlayerTeamHistory를 가지며, 이를 통해 현재 팀과 과거 팀 이력을 표현한다.
- 하나의 Player는 특정 시즌에 하나 이상의 PlayerSeasonStat을 가진다.
- 하나의 Game은 정확히 하나의 Season에 속하며, 두 개의 Team과 연결된다.
- 하나의 Game은 여러 InningScore와 PlayEvent를 가진다.
- 하나의 Game은 여러 PlayerGameStat과 두 개의 TeamGameStat을 가진다.
- 시즌 누적 기록은 경기별 기록의 집계 결과로 관리될 수 있다.
- 고급 지표는 원천 기록을 기반으로 계산하거나 사전 계산된 결과를 별도 계층으로 저장할 수 있다.

## Data Granularity

- 시즌 단위: 선수 기록, 팀 기록, 순위성 데이터의 기본 조회 단위
- 선수 커리어 단위: 한 선수의 데뷔 시즌부터 현재까지 이어지는 시즌별 기록 타임라인
- 팀 시즌 단위: 특정 팀이 한 시즌 동안 쌓아온 누적 기록과 시즌 진행 흐름
- 경기 단위: 경기 결과, 경기별 선수 기록, 경기별 팀 기록의 상세 조회 단위
- 플레이 이벤트 단위: 회차별 점수 변화와 개별 장면을 표현하는 raw 스크래핑 단위
- 일 단위: 시즌 진행 중 하루 1회 스크래핑과 데이터 갱신 실행 단위
- 향후 확장 단위: 예측 모델 입력용 시계열 데이터, 최근 N경기 흐름 데이터

## PostgreSQL Considerations

- 시즌, 팀, 선수, 경기 식별자를 명확히 분리해 정규화 구조를 유지한다.
- 선수 프로필과 시즌 기록, 소속 이력은 분리 저장하는 편이 변경 이력 관리에 유리하다.
- 시즌 누적 기록과 경기 기록은 조회 패턴이 다르므로 별도 테이블로 관리하는 편이 유리하다.
- 기본 기록과 고급 지표는 계산 비용과 갱신 주기를 고려해 원천 테이블과 파생 지표 계층을 나누는 것이 바람직하다.
- 데이터 수집 시 중복 삽입을 방지하기 위해 시즌/경기/선수 기준의 고유 키 설계가 필요하다.
- 일일 갱신 이력을 추적할 수 있도록 수집 시각과 데이터 기준일을 함께 저장한다.
- 향후 예측 기능 확장을 고려해 원천 기록과 가공 지표를 분리하는 방향이 적합하다.

초기 PostgreSQL 테이블 초안은 `docs/data/postgresql-schema-outline.md` 에서 별도로 관리한다.
