# Information Architecture

## Primary Navigation

- 홈: 서비스 소개, 주요 탐색 진입점, 최신 업데이트 상태
- 선수 기록: 시즌별 선수 기록 목록과 상세 조회
- 팀 기록: 시즌별 팀 기록 목록과 상세 조회
- 경기 기록: 경기 일정 기준의 경기별 기록 조회
- 데이터 업데이트 안내: 최근 스크래핑 반영 시점과 데이터 기준 안내

## Key Screens

- 홈 화면: 서비스 목적, 주요 메뉴, 최근 반영 시점, 대표 데이터 진입 링크 제공
- 선수 기록 목록 화면: 시즌, 팀, 선수명 기준으로 선수 기록 탐색
- 선수 상세 화면: 특정 선수의 기본 프로필, 소속 팀 이력, 데뷔 이후 시즌별 기록, 필요 시 경기별 기록과 시즌 간 흐름 제공
- 팀 기록 목록 화면: 시즌 기준 팀 기록 조회 및 팀 간 비교 진입점 제공
- 팀 상세 화면: 특정 팀의 시즌별 누적 기록, 시즌 흐름, 시즌 내 주요 팀 지표 비교 제공
- 경기 기록 목록 화면: 시즌과 날짜 기준으로 경기 목록 탐색
- 경기 상세 화면: 한 경기의 최종 스코어, 회차별 점수, 타석/투구 중심의 raw 이벤트 흐름, 선수별 기록을 가능한 한 원본에 가깝게 제공

## User Flows

- 시즌별 선수 기록 조회
  1. 사용자가 선수 기록 메뉴에 진입한다.
  2. 시즌을 선택한다.
  3. 팀 또는 선수명을 기준으로 목록을 좁힌다.
  4. 선수 상세 화면에서 프로필 정보와 소속 이력을 먼저 확인한다.
  5. 데뷔 시즌부터 현재까지의 시즌별 누적 기록을 확인한다.
  6. 필요 시 경기별 기록 또는 고급 지표 탭으로 이동한다.
- 팀 기록 조회
  1. 사용자가 팀 기록 메뉴에 진입한다.
  2. 시즌을 선택한다.
  3. 팀 목록에서 원하는 팀을 선택한다.
  4. 팀 상세 화면에서 해당 시즌의 누적 팀 기록을 확인한다.
  5. 필요 시 시즌 내 주요 경기 흐름이나 팀 비교 정보로 이동한다.
- 경기 기록 조회
  1. 사용자가 경기 기록 메뉴에 진입한다.
  2. 시즌과 날짜를 선택한다.
  3. 경기 목록에서 특정 경기를 선택한다.
  4. 경기 상세 화면에서 최종 점수와 회차별 점수 흐름을 확인한다.
  5. 타석 결과, 득점 장면, 주요 선수 기록 등 raw 경기 데이터를 확인한다.

## Search and Filter Model

- 공통 기본 필터: 시즌
- 선수 기록 필터: 팀, 선수명
- 선수 상세 기본 구성: 프로필 요약, 팀 이력, 시즌별 기록 테이블, 고급 지표 영역
- 팀 기록 필터: 시즌, 팀
- 경기 기록 필터: 시즌, 날짜, 팀
- 경기 상세 기본 구성: 경기 메타 정보, 스코어보드, 회차별 기록, 선수별 경기 기록, raw 이벤트 흐름
- 기본 탐색 원칙: 사용자는 시즌을 먼저 정하고, 이후 팀/선수/경기 단위로 범위를 좁히는 흐름을 사용한다.
